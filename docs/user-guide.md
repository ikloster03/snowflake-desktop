# Руководство пользователя Snowflake Desktop

## Введение

## Начало работы

### Системные требования

- Операционная система: Windows 10/11, Linux Ubuntu 20.04+
- Оперативная память: минимум 4 ГБ
- Свободное место на диске: минимум 100 МБ

### Установка

1. Скачайте установщик с [официальной страницы загрузки](/downloads)
2. Запустите установщик и следуйте инструкциям
3. После установки запустите приложение

### Первый запуск

При первом запуске приложение создаст рабочую папку для ваших проектов. Все данные хранятся локально на вашем компьютере, обеспечивая полную конфиденциальность.

## Управление проектами

### Создание нового проекта

1. На главном экране нажмите кнопку "Новый проект"
2. Введите название проекта
3. Выберите папку для сохранения проекта
4. Нажмите "Создать"

### Открытие существующего проекта

1. На главном экране выберите "Открыть проект"
2. Найдите нужную папку проекта
3. Выберите папку и нажмите "Открыть"

### Настройки проекта

В разделе "Настройки проекта" вы можете:

- Изменить название проекта
- Управлять метаданными проекта

## Работа с книгами

### Создание новой книги

1. В модуле "Книги" нажмите "Новая книга"
2. Заполните основные поля:
   - Название
   - Автор(ы)
   - Жанр(ы)
   - Описание
3. Нажмите "Сохранить"

### Управление метаданными книги

Для каждой книги вы можете указать:
- Название и подзаголовок
- Автора или соавторов
- Жанр и поджанры
- Краткое описание
- Статус работы (черновик, в работе, завершено)
- Целевая аудитория
- Планируемый объем

## Управление лором книги

### Персонажи

#### Создание персонажа

1. В модуле "Лор" выберите вкладку "Персонажи"
2. Нажмите "Новый персонаж"
3. Заполните профиль персонажа:
   - Имя и фамилия
   - Возраст
   - Внешность
   - Характер и личность
   - Биография и предыстория
   - Роль в сюжете
4. Нажмите "Сохранить"

#### Связи между персонажами

Вы можете создавать различные типы связей между персонажами:
- Семейные отношения
- Дружба
- Романтические отношения
- Профессиональные связи
- Конфликты и вражда

### Места

#### Создание локации

1. В модуле "Лор" выберите вкладку "Места"
2. Нажмите "Новое место"
3. Заполните описание места:
   - Название локации
   - Тип места (город, здание, природная зона и т.д.)
   - Подробное описание
   - Атмосфера и настроение
   - Важные особенности
   - Связанные персонажи
4. Нажмите "Сохранить"

#### Связи между местами

Создавайте связи между локациями:
- Географическое расположение
- Пути перемещения
- Административное подчинение
- Историческая связь

### Вещи

#### Создание предмета

1. В модуле "Лор" выберите вкладку "Вещи"
2. Нажмите "Новая вещь"
3. Заполните описание предмета:
   - Название предмета
   - Тип (оружие, артефакт, документ и т.д.)
   - Внешний вид
   - Свойства и особенности
   - История создания
   - Текущий владелец
   - Значение для сюжета
4. Нажмите "Сохранить"

### События

#### Создание события

1. В модуле "Лор" выберите вкладку "События"
2. Нажмите "Новое событие"
3. Заполните описание события:
   - Название события
   - Дата или временной период
   - Место происшествия
   - Участники события
   - Подробное описание
   - Причины и последствия
   - Влияние на сюжет
4. Нажмите "Сохранить"

### Визуализация связей

Используйте интерактивную карту лора для:
- Просмотра всех элементов лора
- Анализа связей между персонажами, местами, вещами и событиями
- Поиска потенциальных конфликтов
- Планирования развития мира
- Отслеживания хронологии событий

## Планирование сюжета

### Создание структуры книги

1. В модуле "Планирование" выберите нужную книгу
2. Создайте главы, нажав "Добавить главу"
3. Для каждой главы:
   - Введите название
   - Добавьте краткое описание
   - Напишите содержание в текстовом редакторе
   - Добавьте сцены при необходимости

### Работа с главами

Для каждой главы доступны следующие функции:
- Редактирование текста с автосохранением
- Добавление и управление сценами
- Отслеживание количества слов
- Пометки о статусе (черновик, готово, требует правки)

### Управление сценами

Для каждой сцены можно указать:
- Название и описание
- Участвующих персонажей
- Локацию действия
- Цель сцены
- Конфликт или напряжение

## Экспорт и сохранение

### Автоматическое сохранение

Проект автоматически сохраняется каждые 30 секунд. Все изменения сохраняются локально на вашем компьютере.

### Экспорт в DOCX

1. В главном меню выберите "Экспорт"
2. Выберите место сохранения
3. Нажмите "Экспортировать"

Текст будет экспортирован в формате Microsoft Word (.docx) с сохранением форматирования.

### Резервное копирование

Рекомендуется регулярно создавать резервные копии:
1. Скопируйте папку проекта в безопасное место
2. Используйте облачное хранилище для дополнительной защиты
3. Создавайте копии перед крупными изменениями

## Часто задаваемые вопросы

### Где хранятся мои данные?

Все данные хранятся локально на вашем компьютере в папке проекта. Никакая информация не передается в интернет.

### Можно ли работать без интернета?

Да, Snowflake Desktop полностью работает офлайн. Интернет не требуется для использования приложения.

### Как перенести проект на другой компьютер?

Скопируйте папку проекта на другой компьютер и откройте ее через "Открыть проект".

### Есть ли ограничения на размер проекта?

Практических ограничений нет. Приложение оптимизировано для работы с большими проектами.

### Как восстановить случайно удаленный текст?

Используйте функцию "Отменить" в редакторе или восстановите из резервной копии проекта.

### Поддерживается ли совместная работа?

В текущей версии совместная работа не поддерживается. Каждый проект предназначен для работы одного автора.

### В каких форматах можно экспортировать текст?

В настоящее время поддерживается экспорт только в формат DOCX (Microsoft Word).
